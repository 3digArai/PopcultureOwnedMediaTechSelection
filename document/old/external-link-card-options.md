# 外部リンク紹介の表示方式メモ

## 目的
- 自分で記事を全て書かない場合でも、外部の良質な記事を見やすく紹介できるようにする。

## 主な選択肢

## OGP自動カード
- URLを入力すると、`title / description / image / siteName` を取得してカード表示する方式。
- 最も定番で、編集負荷と見た目のバランスが良い。
- 取得失敗時のフォールバック（タイトルのみ表示など）を用意する必要がある。

## 埋め込み（oEmbed / iframe）
- YouTube、Xなど対応サービスをリッチに表示する方式。
- 見た目は強いが、対応サイトが限定される。
- レイアウト崩れや読み込み遅延に注意が必要。

## 手動カード
- URLに加えて、見出し・説明文・画像を編集者が入力する方式。
- 品質を安定させやすいが、運用コストが上がる。
- OGP取得失敗時の代替として有効。

## 外部API利用（Microlink / Iframely等）
- メタ情報取得と整形を外部サービスに委譲する方式。
- 実装は早いが、外部依存・コスト・利用制限が増える。

## 推奨方針（暫定）
- 基本は `OGP自動カード`
- 取得失敗時は `手動カード` にフォールバック
- 特定サービス（動画/SNS）は `埋め込み` を併用

## 実装時の注意点
- 外部サイト本文の転載は避ける（引用ルールを明確化）
- OGP画像の扱いは権利面を確認する
- クローリング失敗時の表示劣化を設計しておく
- キャッシュ期間を設定して、毎回取得しないようにする

## 現在の候補構成との相性
- WordPress: プラグインで対応しやすい
- Headless + Next/Astro: サーバー側でOGP取得APIを実装しやすい
- 自作CMS: 完全自由だが、取得・キャッシュ・障害時対応も自前になる
