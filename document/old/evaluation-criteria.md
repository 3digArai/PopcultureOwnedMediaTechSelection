# 技術選定 評価観点一覧

## 目的
- オウンドメディア/ブランドサイトの技術選定を、引き継ぎ可能な判断材料として残す。

## 評価観点（一覧）

| 評価観点 | 何を見るか | 高評価の目安 | 低評価の目安 |
|---|---|---|---|
| 運用継続性 | 日常更新・保守を無理なく続けられるか | 更新担当が迷わず回せる | 更新手順が複雑で止まりやすい |
| 引き継ぎ容易性 | 担当交代時に運用を再現できるか | ドキュメントと手順で再現可能 | 個人依存で暗黙知が多い |
| 編集体験 | 非エンジニアが記事作成しやすいか | 管理画面で完結しやすい | コード変更が頻発する |
| 表現自由度 | ブランド表現・UIをどこまで作り込めるか | 制約が少なく実装自由度が高い | テンプレート制約が強い |
| 表示性能 | ページ表示速度・快適性 | 主要ページの表示が軽い | 初回表示が重く離脱しやすい |
| SEO対応力 | メタ情報・構造化・URL設計を制御できるか | SEO要件を実装で担保しやすい | プラットフォーム制約が大きい |
| セキュリティ運用 | 脆弱性対応や更新運用のしやすさ | 保守手順が単純で更新しやすい | プラグイン/依存更新が複雑 |
| 拡張性 | 将来機能（会員、検索、多言語等）に対応しやすいか | 機能追加時の制約が少ない | 追加時に全面改修になりやすい |
| プラグイン/拡張エコシステム | プラグイン等で必要機能を短期間に追加できるか | 実績ある拡張が豊富で導入が容易 | 要件に合う拡張が少なく自作が多い |
| コスト（初期） | 立ち上げに必要な開発・制作コスト | 小さく始めやすい | 初期設計/実装コストが大きい |
| コスト（運用） | 月額費用と保守工数のバランス | 固定費と運用工数が読みやすい | 継続コストが膨らみやすい |
| 日本での使いやすさ | 日本語情報・国内事例・国内パートナーの見つけやすさ | 日本語情報が豊富で外注/採用もしやすい | 情報が少なく運用時の相談先が限られる |
| 採用/外注しやすさ | 人材確保・外部パートナー連携のしやすさ | 対応できる人材が市場に多い | 特殊スキル依存が強い |
| ベンダーロックイン | 特定サービス依存による移行難易度 | データ移行・再構築の道筋がある | 乗り換え時の再構築負担が大きい |

## 判定ルール（推奨）
- 評価は `5段階（5:非常に良い - 1:弱い）` で統一する。
- 観点ごとに `重み（1-5）` を設定し、総合点は `評価点 × 重み` の合計で算出する。
- 総合点だけで決めず、`必須条件`（例: 引き継ぎ容易性3以上）を先に満たすか確認する。

## 必須条件の例
- 引き継ぎ容易性: 3以上
- 運用継続性: 3以上
- 編集体験: 3以上
- コスト（運用）: 3以上

## 補足
- ブランド表現を重視する場合は `表現自由度` と `表示性能` の重みを上げる。
- 少人数運用を重視する場合は `運用継続性` と `編集体験` の重みを上げる。
- 将来の機能追加が不確実な場合は `拡張性` と `ベンダーロックイン` を評価に残す。
