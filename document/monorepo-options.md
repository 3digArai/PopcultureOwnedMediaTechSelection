# モノレポ運用構成候補（比較用）

## 目的
- `evaluation-criteria.md` の評価軸で、モノレポ運用構成候補を比較する。

## 候補一覧（モノレポ運用）

| 構成 | 区分 | 向いているケース | メモ |
|---|---|---|---|
| pnpm Workspaces + Changesets | パッケージ運用 | 複数サイトを堅実に運用したい | シンプルで運用説明がしやすい。 |
| Turborepo + Next/Astro | ビルド最適化 | 複数アプリを高速に開発したい | 実務採用が多く再利用設計に向く。 |
| Nx + Next/Nuxt | 統合開発基盤 | 大規模チームで規約統制したい | 高機能だが学習コストがある。 |
| Lerna + Yarn Workspaces | 既存保守型 | 既存レガシーを維持しながら運用 | 新規採用では優先度は低め。 |

## 先に比較に載せる推奨候補

1. pnpm Workspaces + Changesets
2. Turborepo + Next/Astro
3. Nx + Next/Nuxt
4. Lerna + Yarn Workspaces

## 評価前提

- スコア: `5（非常に良い）- 1（弱い）`
- 想定: ポップカルチャー紹介のブランドメディアを、少人数で継続運用するケース。

## 候補別 採点理由（観点別）

### pnpm Workspaces + Changesets
- 運用継続性 `4`: 構成が比較的単純で運用負荷を抑えやすい。
- 引き継ぎ容易性 `4`: ルールが明確で引き継ぎしやすい。
- 編集体験 `3`: 編集体験への直接効果は中程度。
- 表現自由度 `4`: フロント構成の自由度を維持できる。
- 表示性能 `4`: 実装品質次第で十分な性能。
- SEO対応力 `4`: アプリ側設計で対応可能。
- セキュリティ運用 `4`: 依存管理を集中化しやすい。
- 拡張性 `4`: 複数プロジェクトの拡張に向く。
- プラグイン/拡張エコシステム `3`: 必要十分なツール資産。
- コスト（初期） `3`: 導入は中程度。
- コスト（運用） `4`: 長期的なメンテ性が高い。
- 日本での使いやすさ `3`: 国内情報は比較的豊富。
- 採用/外注しやすさ `3`: Node系人材で対応しやすい。
- ベンダーロックイン `4`: OSS中心で可搬性が高い。

### Turborepo + Next/Astro
- 運用継続性 `4`: ビルド効率が高く運用しやすい。
- 引き継ぎ容易性 `3`: 仕組み理解が必要。
- 編集体験 `3`: 編集体験に直接は効かない。
- 表現自由度 `5`: 複数フロントで高い自由度を維持できる。
- 表示性能 `4`: 最適化パイプラインを作りやすい。
- SEO対応力 `4`: アプリ側制御で対応可能。
- セキュリティ運用 `3`: 依存更新管理の体制は必要。
- 拡張性 `5`: スケールしやすい構成。
- プラグイン/拡張エコシステム `4`: 周辺エコシステムが豊富。
- コスト（初期） `2`: 初期設計は重め。
- コスト（運用） `3`: 運用効率は高いが管理コストはある。
- 日本での使いやすさ `3`: 国内情報は一定ある。
- 採用/外注しやすさ `3`: 対応人材は確保可能。
- ベンダーロックイン `3`: 構成依存はあるが移行余地はある。

### Nx + Next/Nuxt
- 運用継続性 `4`: 規約統制しやすく大規模運用に強い。
- 引き継ぎ容易性 `3`: ルールが厳密で学習コストがある。
- 編集体験 `3`: 編集導線への直接効果は中程度。
- 表現自由度 `5`: アプリ分割運用で自由度を維持可能。
- 表示性能 `4`: 最適化しやすい。
- SEO対応力 `4`: 実装で十分対応可能。
- セキュリティ運用 `3`: 依存管理を厳格運用する必要。
- 拡張性 `5`: 大規模拡張に強い。
- プラグイン/拡張エコシステム `4`: 高機能な運用ツール群がある。
- コスト（初期） `2`: 初期導入が重い。
- コスト（運用） `3`: 長期では効率化できるが学習が必要。
- 日本での使いやすさ `2`: 国内事例は多くない。
- 採用/外注しやすさ `2`: 経験者の母数は限定的。
- ベンダーロックイン `3`: ツール依存はあるが移行可能。

### Lerna + Yarn Workspaces
- 運用継続性 `3`: 既存保守には使えるが新規最適ではない。
- 引き継ぎ容易性 `3`: 古い実装資産として引き継ぎは可能。
- 編集体験 `2`: 編集運用への寄与は小さい。
- 表現自由度 `3`: 必要最低限。
- 表示性能 `3`: 直接効果は限定的。
- SEO対応力 `3`: アプリ設計次第。
- セキュリティ運用 `3`: 依存管理の改善余地がある。
- 拡張性 `3`: 拡張は可能だが最新構成より弱い。
- プラグイン/拡張エコシステム `2`: モダン運用資産は少なめ。
- コスト（初期） `3`: 既存活用なら低め。
- コスト（運用） `3`: 既存維持コストは中程度。
- 日本での使いやすさ `2`: 新規導入情報は少ない。
- 採用/外注しやすさ `2`: 最新案件の経験者は少なめ。
- ベンダーロックイン `3`: 完全固定ではない。
