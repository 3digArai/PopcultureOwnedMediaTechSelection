# 定番CMS候補（WordPress含む）

## 目的
- `evaluation-criteria.md` の評価軸で、定番CMS候補を比較するための候補整理。

## 候補一覧（定番CMS）

| CMS | 区分 | 向いているケース | メモ |
|---|---|---|---|
| WordPress | 伝統的CMS（OSS） | 記事更新が多い、編集者主体、引き継ぎ重視 | 国内外の導入実績が多く、運用人材を確保しやすい。 |
| Movable Type | 伝統的CMS（商用） | 企業サイト、ガバナンス重視、国内運用 | 日本での導入実績が多く、法人運用で使われやすい。 |
| Drupal | 伝統的CMS（OSS） | 大規模サイト、権限設計が複雑な案件 | 柔軟性が高い反面、実装難易度は高め。 |
| Joomla! | 伝統的CMS（OSS） | 中規模メディア、比較的標準的なCMS運用 | WordPressより採用は少ないが歴史ある選択肢。 |
| Ghost | メディア特化CMS（OSS/ホスティング） | 記事配信中心、シンプルな編集運用 | 出版系に強く、UIが軽量。 |
| Craft CMS | 伝統的CMS（商用） | ブランド表現重視、柔軟なコンテンツ設計 | 実装自由度が高く、制作会社案件で採用される。 |
| concrete CMS | 伝統的CMS（OSS） | ページ編集を現場で回したいサイト | 管理画面でのページ編集がしやすい。 |
| TYPO3 | 伝統的CMS（OSS） | 多言語・多サイトを厳格に運用する案件 | エンタープライズ寄りで運用設計が前提。 |
| Adobe Experience Manager (AEM) | エンタープライズCMS（商用） | 大企業、複数ブランド横断、統制重視 | 高機能だが導入・運用コストは大きい。 |
| Sitecore | エンタープライズCMS（商用） | パーソナライズ重視、統合マーケ基盤 | 高機能だが専門人材と予算が必要。 |
| HubSpot CMS Hub | SaaS CMS（商用） | マーケ連携重視、運用をSaaSで完結したい | MA/CRMとの連携が強み。 |

## 先に比較に載せる推奨候補

1. WordPress
2. Movable Type
3. Drupal
4. Ghost
5. Craft CMS
6. HubSpot CMS Hub

## 評価前提

- スコア: `5（非常に良い）- 1（弱い）`
- 想定: ポップカルチャー紹介のブランドメディアを、少人数で継続運用するケース。

## 候補別 採点理由（観点別）

### WordPress
- 運用継続性 `5`: 更新担当向けの管理UIと運用実績が豊富で、日常更新フローを作りやすい。
- 引き継ぎ容易性 `5`: 日本語ドキュメント・既存運用ノウハウが多く、担当交代時の再現性が高い。
- 編集体験 `4`: ブロックエディタで十分な編集ができる一方、複雑レイアウトは調整が必要。
- 表現自由度 `4`: テーマ/カスタム実装で広く対応できるが、完全自由度はヘッドレス構成に劣る。
- 表示性能 `3`: 構成次第で改善可能だが、プラグイン過多や重いテーマで劣化しやすい。
- SEO対応力 `5`: メタ/構造化/サイトマップなど実装手段が多く、運用で補正しやすい。
- セキュリティ運用 `3`: コア/テーマ/プラグイン更新を継続管理する必要があり、運用設計が前提。
- 拡張性 `5`: カスタム投稿・API・会員機能などの拡張手段が豊富。
- プラグイン/拡張エコシステム `5`: 必要機能を短期間で追加しやすく、選択肢も多い。
- コスト（初期） `5`: テンプレート・既存拡張を活用すれば立ち上げコストを抑えやすい。
- コスト（運用） `4`: SaaSよりランニングは抑えやすいが、保守工数は一定必要。
- 日本での使いやすさ `5`: 国内事例・制作会社・運用担当人材が多く、相談先を確保しやすい。
- 採用/外注しやすさ `5`: 対応人材の母数が多く、外注先選定もしやすい。
- ベンダーロックイン `4`: OSSで移行余地はあるが、テーマ/プラグイン依存が強いと移行負荷は残る。

### Movable Type
- 運用継続性 `4`: 企業運用の定型フローを組みやすく、公開運用の安定性が高い。
- 引き継ぎ容易性 `4`: 国内法人利用の蓄積があり、運用手順を標準化しやすい。
- 編集体験 `3`: 基本編集は可能だが、直感的UIではWordPress系に見劣りする。
- 表現自由度 `3`: 実装可能だが、モダンなフロント要件では工夫が必要。
- 表示性能 `3`: 運用設計次第で安定するが、構成の自由度は限定される。
- SEO対応力 `4`: URL設計やメタ対応は行いやすく、運用型SEOに対応可能。
- セキュリティ運用 `4`: 法人運用での管理しやすさがあり、保守統制を取りやすい。
- 拡張性 `4`: 追加実装は可能だが、対応可能な拡張の選択肢は限定的。
- プラグイン/拡張エコシステム `3`: 実績ある拡張はあるが、WordPressほど裾野は広くない。
- コスト（初期） `3`: ライセンスや設計を含めると初期負担は中程度。
- コスト（運用） `3`: 安定運用しやすい一方、商用前提の費用と保守工数がかかる。
- 日本での使いやすさ `4`: 国内情報と導入事例が多く、日本語運用に馴染みがある。
- 採用/外注しやすさ `4`: 国内制作会社の対応はあるが、母数はWordPressより少ない。
- ベンダーロックイン `3`: 仕組みに乗るほど移行時の再設計負担が発生しやすい。

### Drupal
- 運用継続性 `3`: 仕組み化すれば安定するが、運用側に一定の技術理解が必要。
- 引き継ぎ容易性 `3`: 設計思想の理解が必要で、引き継ぎ時の学習コストが高め。
- 編集体験 `3`: 標準機能で運用は可能だが、編集UXは構築次第で差が出る。
- 表現自由度 `5`: コンテンツモデルと権限設計の自由度が高く、大規模要件に強い。
- 表示性能 `4`: 適切な構成で高い性能を出しやすい。
- SEO対応力 `4`: SEO実装は十分可能で、要件に応じたチューニングができる。
- セキュリティ運用 `4`: 企業向け運用実績があり、統制を前提に安全運用しやすい。
- 拡張性 `5`: 複雑要件・多言語・ワークフローなど拡張耐性が高い。
- プラグイン/拡張エコシステム `4`: モジュール資産は多いが、選定・実装の難易度は高い。
- コスト（初期） `2`: 設計/実装工数が大きく、小規模案件では過剰投資になりやすい。
- コスト（運用） `3`: 安定運用は可能だが、保守できる体制の維持コストが必要。
- 日本での使いやすさ `3`: 国内情報・事例はあるが、WordPressほどの厚みはない。
- 採用/外注しやすさ `3`: 対応できる会社/人材は限定的で調達難易度が上がる。
- ベンダーロックイン `4`: OSSで移行余地はあるが、作り込み量が多いほど移行負荷は増える。

### Ghost
- 運用継続性 `4`: 記事配信中心なら運用がシンプルで回しやすい。
- 引き継ぎ容易性 `3`: 仕組み自体は明快だが、日本語での引き継ぎ資料は限定的。
- 編集体験 `4`: 執筆中心の編集体験は良好で、運用の認知負荷が低い。
- 表現自由度 `4`: テーマ実装で表現は可能だが、用途特化ゆえ制約もある。
- 表示性能 `4`: 軽量構成を保ちやすく、表示性能を確保しやすい。
- SEO対応力 `4`: 基本SEO機能は揃っており、記事運用には十分対応できる。
- セキュリティ運用 `4`: 構成が比較的シンプルで、保守対象を絞りやすい。
- 拡張性 `3`: メディア運用には十分だが、複雑機能追加には限界がある。
- プラグイン/拡張エコシステム `3`: エコシステムはあるが、WordPress系より選択肢は少ない。
- コスト（初期） `4`: 記事中心であれば比較的短期間で立ち上げられる。
- コスト（運用） `4`: シンプル運用なら工数を抑えやすい。
- 日本での使いやすさ `2`: 国内事例・日本語情報・支援会社が少なく、導入判断が難しい。
- 採用/外注しやすさ `2`: 国内での対応人材が少なく、外注選択肢が限定される。
- ベンダーロックイン `4`: OSS基盤で可搬性はあるが、テーマ/運用設計依存は残る。

### Craft CMS
- 運用継続性 `3`: 丁寧に設計すれば回るが、運用側にも設計理解が必要。
- 引き継ぎ容易性 `3`: 実装思想への依存が強く、引き継ぎにドキュメント整備が必須。
- 編集体験 `3`: 設計次第で良くなるが、標準状態での分かりやすさは中程度。
- 表現自由度 `5`: 柔軟なコンテンツモデリングでブランド表現を作り込みやすい。
- 表示性能 `4`: 適切実装で高パフォーマンスを維持しやすい。
- SEO対応力 `4`: SEO要件を実装レベルで調整しやすい。
- セキュリティ運用 `4`: 管理対象を明確にしやすく、保守運用を設計しやすい。
- 拡張性 `4`: 追加開発で拡張しやすいが、実装工数は相応に必要。
- プラグイン/拡張エコシステム `3`: 拡張手段はあるが、規模はWordPressより小さい。
- コスト（初期） `2`: 設計・開発前提になりやすく、初期費用が高くなりやすい。
- コスト（運用） `3`: 運用は安定させられるが、開発保守を継続する前提。
- 日本での使いやすさ `2`: 日本語情報と国内導入事例が少なく、検討コストが高い。
- 採用/外注しやすさ `2`: 対応できる人材・会社の選択肢が限られる。
- ベンダーロックイン `3`: 実装資産の再利用は可能だが、移行時は再設計が必要。

### HubSpot CMS Hub
- 運用継続性 `4`: SaaSとして基盤運用を委譲でき、日常運用を標準化しやすい。
- 引き継ぎ容易性 `4`: 画面ベースで運用しやすく、手順化しやすい。
- 編集体験 `4`: マーケ運用を前提とした編集体験で、担当者運用に向く。
- 表現自由度 `3`: 一般的な要件は対応できるが、自由度はフルスクラッチより低い。
- 表示性能 `3`: SaaS基盤で安定するが、細かな最適化自由度は限定される。
- SEO対応力 `4`: マーケ機能と一体でSEO運用しやすい。
- セキュリティ運用 `5`: インフラ/基盤保守をサービス側に寄せられる。
- 拡張性 `4`: 連携機能は豊富だが、サービス仕様の枠内での拡張が中心。
- プラグイン/拡張エコシステム `4`: 連携アプリは多いが、OSS系ほど自由度はない。
- コスト（初期） `3`: 立ち上げはしやすいが、運用設計/既存資産移行の調整が必要。
- コスト（運用） `2`: サブスク費用が継続し、規模拡大で負担増になりやすい。
- 日本での使いやすさ `3`: 日本語情報はあるが、CMS専業製品ほど事例は多くない。
- 採用/外注しやすさ `3`: 対応パートナーはあるが、一般CMSより母数は少ない。
- ベンダーロックイン `2`: プラットフォーム依存が強く、移行時の再構築負荷が高い。
